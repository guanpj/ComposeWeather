# Compose Weather Web 版本说明

## ⚠️ 已知问题：中文字符显示

### 问题描述
Web 版本中，中文字符和特殊符号（如 ℃）可能显示为乱码或方框。

### 原因
Compose for Web 使用 **Skiko WASM** 渲染引擎（基于 Skia），默认情况下不包含中文字体文件。这是一个已知的限制。

### 解决方案

#### 方案 1: 使用浏览器扩展（临时解决）
1. 安装 **Stylus** 或类似的 CSS 注入扩展
2. 为页面添加自定义 CSS 强制使用系统字体

#### 方案 2: 等待官方支持
Compose Multiplatform for Web 正在积极开发中，未来版本可能会改进字体支持。

#### 方案 3: 修改为英文版本（最简单）
如果这是演示项目，可以将中文文本改为英文：
- "深圳市" → "Shenzhen"
- "℃" → "°C" (使用 ASCII 度数符号)
- "风向" → "Wind"
- "湿度" → "Humidity"
等等

### 推荐做法
目前建议使用 **Android、iOS 或 Desktop** 版本来体验完整功能。Web 版本作为技术演示，主要验证跨平台能力。

## 🎯 功能状态

| 功能 | 状态 | 说明 |
|------|------|------|
| Net 网络加载 | ✅ 正常 | 可以获取天气数据 |
| Cache 缓存 | ❌ 不支持 | 点击会显示 "no cache" |
| 中文显示 | ⚠️ 问题 | Skiko WASM 限制 |
| 图标显示 | ✅ 正常 | PNG 图标正常加载 |
| 布局渲染 | ✅ 正常 | UI 布局正确 |

## 📝 使用建议

1. **开发测试**: 使用 Android/iOS/Desktop 版本
2. **演示展示**: 如果只展示 UI 布局和交互，Web 版本可用
3. **生产使用**: 推荐原生平台版本

## 🔧 技术细节

- **渲染引擎**: Skiko WASM (Skia for Kotlin)
- **字体加载**: 目前不支持动态字体加载
- **编码设置**: UTF-8 (已正确配置)
- **HTML 字体**: 已配置中文字体回退，但 Canvas 渲染不使用 CSS
